Disjoint-Set (Union-Find)

The Disjoint-Set data structure, also known as Union-Find, is designed to efficiently keep track of a partition of a set into disjoint subsets. It supports two main operations: union and find

  • Union: Merges two subsets into a single set.
  • Find: Determines the subset to which a particular element belongs and can be used to check if two elements are in the same subset

This data structure is particularly useful for solving dynamic connectivity, such as determening whether there is a path between two elements in a graph.

How it works:

  1. Each element initially belongs to its own subset, and a representative (root) is assigned to each subset.
  2. The 'Union' operation merges two subsets by making the root of one subset to the root of the other subset
  3. The 'Find' operation determines the representative of the subset to which an element belongs by following the chain of parent pointers until reaching the root.
